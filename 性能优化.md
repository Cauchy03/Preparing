## 1. 前端性能优化指标RAIL

```
RAIL是一组前端性能指标，代表响应（Response）、动画（Animation）、空闲（Idle）、加载（Load）
响应：指从用户触发某个操作到系统对该操作做出的响应时间
动画：指动画的流程度，也就是每秒帧数
空闲：指浏览器在空闲状态下执行一些必要的操作，如预加载资源
加载：指页面加载的总时间，包括网络请求时间、解析HTML、执行JS、渲染等
```

## 3. 重排和重绘

```
重排和重绘是浏览器关键渲染路径上的两个节点， 浏览器的关键渲染路径就是 DOM树 和 CSS树 生成渲染树，然后根据渲染树进行布局(layout)来确定页面上内容的大小和位置，最后将像素绘制 (Paint)到屏幕上。
其中重排就是当元素的位置发生变动的时候，浏览器重新执行布局这个步骤，来重新确定页面上内容的大小和位置，确定完之后就会进行重新绘制到屏幕上，所以重排一定会导致重绘。导致重排的原因可能是修改页面元素尺寸、位置、字体大小、浏览器窗口等。
如果元素位置没有发生变动，仅仅只是样式发生变动，没有影响布局，这个时候浏览器重新渲染的时候会跳过布局步骤，直接进入绘制步骤，这就是重绘，所以重绘不一定会导致重排。
```

## 6. 描述下浏览器从输入网址到页面展现的整个过程

```
首先会进行一个DNS解析，将输入的信息转换成对应的ip地址，建立tcp连接，这时候要看是http还是https，如果是https还会进行SSR套阶层，主要是为了更好的SEO和安全，然后会进行三次握手四次挥手，具体四次挥手的时间要看有没有设置keep alive属性，如果没有，所有的请求都会口进行三次握手四次挥手，就非常浪费时间，但http1.0过后默认都会加这个属性，当我们所有访问http请求之后它才进行这个四次挥手。所有数据都会经过网络五层模型，然后拉数据，如果是单页面开发，他会首先拉html文档，生成dom，加载到css生成css树，dom树和css树结合生成渲染树，如果遇到js的话，无论是下载还是解析，都会阻塞整个文档的渲染，js内部执行又分为同步任务和异步任务，同步任务从上到下一次执行，碰到异步任务，就会将异步任务放到宿主环境中可以是node或者浏览器，当合适的时机在放到异步回调队列里，等到执行栈为空，就放到执行栈中执行，异步任务又分为微任务和宏任务，像promise就是微任务，微任务在宏任务之前执行
```
